# E資格 学習ナビ

JDLA E資格の私用学習支援アプリ(PWA)。間隔反復・自信度/誤答原因の記録・用語/数式カード・コード読解・カメラ画像解析ラボを、依存ライブラリゼロの静的ファイルだけで提供する。

- 学習履歴はブラウザの IndexedDB にのみ保存する(このリポジトリには含めない)
- カメラ映像は端末内の Canvas で処理し、保存・送信しない
- GitHub Actions・ビルド工程は使わない(main の root をそのまま公開)

## 構成

```
/
├── index.html           アプリ本体(単一ファイル)
├── manifest.webmanifest PWAマニフェスト
├── sw.js                Service Worker(オフラインキャッシュ)
├── icons/               アプリアイコン
│   ├── icon-192.png
│   ├── icon-512.png
│   ├── maskable-512.png
│   └── apple-touch-icon.png
└── README.md
```

## GitHub Pages の有効化手順

1. GitHub で Public リポジトリを作成する(例: `eshikaku-navi`)
2. 上記ファイル一式を **main ブランチの root** に配置(push)する
3. リポジトリの **Settings → Pages** を開く
4. **Build and deployment** の Source を **Deploy from a branch** にする
5. Branch を **main** / フォルダを **/(root)** にして **Save**
6. 数分待つと `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開される
7. 公開URLを開き、記録タブの「アプリ版」表示と、ホーム画面が出ることを確認する

※ カメラ機能(画像解析ラボ)は HTTPS が必須のため、`file://` では動かず GitHub Pages 上で動作する。初回のカメラ起動時にブラウザの許可ダイアログが出る。

## iPhone でのホーム画面追加手順(PWAとして使う)

1. **Safari** で公開URLを開く(Chrome等ではなく Safari を使う)
2. 画面下中央の **共有ボタン(四角から上矢印)** をタップ
3. メニューから **「ホーム画面に追加」** をタップ
4. 名前(E資格ナビ)を確認して **「追加」** をタップ
5. ホーム画面のアイコンから起動すると、アドレスバーなしの全画面アプリとして動く

補足:
- iOS の IndexedDB は「ホーム画面追加後のPWA」と「Safariのタブ」で保存領域が別になる。**学習はどちらか一方(PWA推奨)に固定**すること
- 端末を変えるときは、記録タブの「学習データを書き出す(JSON)」→ 新端末で「読み込む」で移行する

## Service Worker のキャッシュ更新ルール(重要)

`sw.js` の先頭にある `CACHE_NAME` がキャッシュの世代を決めている。

```js
const CACHE_NAME = "eshikaku-v0.2.0";
```

**ルール**

1. `index.html` / `manifest.webmanifest` / `icons/` のいずれかを変更して push するときは、**必ず `CACHE_NAME` の末尾バージョンを1つ上げる**(例: `v0.2.0` → `v0.2.1`)
2. あわせて `index.html` 末尾の `APP_VERSION` も同じ値に更新する(記録タブに表示され、更新確認に使える)
3. 版の付け方の目安
   - 見た目・文言・軽微修正: パッチを上げる(`v0.2.0` → `v0.2.1`)
   - 機能追加: マイナーを上げる(`v0.2.x` → `v0.3.0`)

**仕組み(古い版が残りにくい理由)**

- ページ本体はネットワーク優先で取得するため、オンラインなら常に最新の `index.html` が届く
- 新しい `sw.js` は `skipWaiting()` で即時有効化され、`activate` 時に **`CACHE_NAME` が一致しない旧キャッシュをすべて削除**する
- アプリ側は SW の切り替わりを検知して自動再読み込みする(学習セッション中は自動再読み込みせず案内のみ)

**更新が反映されないと感じたとき**

1. 記録タブの「アプリ版」が新しい値になっているか確認する
2. ページを一度閉じて開き直す(PWAはタスクからスワイプで終了→再起動)
3. それでも古い場合はブラウザの再読み込みを2回行う

## リポジトリに含めないもの

- 学習履歴のバックアップJSON(`eshikaku_backup_*.json`)
- 講義資料・市販教材などの著作物
- APIキー・認証情報
- 個人データ

`.gitignore` でバックアップJSONを除外している。教材由来のテキストを問題JSONに含める場合は、利用規約・著作権条件を確認してから行うこと。
