# v0.4.0 学習データ読込手順

対象版: `v0.4.0-dev.19`

## 読込方法

記録画面の「データ管理」で、JSONを選ぶ前に読込方法を指定します。

### 統合読込（新しい状態を優先）

通常はこちらを使用します。

- JSON内の問題を追加・更新
- 回答ログを追加・更新
- 問題の復習予定は更新日時が新しい側を採用
- カード理解度は`updatedAt`が新しい側を採用
- 現在の新しい状態は古いJSONで上書きしない
- 読込前に端末内の復元用スナップショットを作成

端末間のデータ集約や、日常的なバックアップ読込に向いています。

### 学習状態を置換（JSONへ戻す）

過去のJSONに保存された理解度・回答状態へ戻す場合に使用します。

置換対象:

- 回答履歴
- 問題の復習予定
- カード理解度と次回復習日

保持するもの:

- 現在の組込み問題教材
- 現在のカード教材
- JSONに含まれない現在の問題教材

JSON内の問題は追加・更新しますが、現在の問題ストアは全削除しません。このため、古いJSONを読み込んでもv0.4.0の組込み174問を失いません。

## 置換読込の安全策

置換前に、次の2つを自動作成します。

1. 現在の学習データJSON
2. 端末内の復元用スナップショット

自動保存ファイル名:

```text
E資格学習ナビ_学習データ_YYYYMMDD_HHMMSS.json
```

確認画面には、現在とJSONの次の件数を表示します。

```text
問題
回答
問題復習
カード理解度
```

確認画面でキャンセルした場合、データは変更しません。

## 旧形式JSON

`cardProgress`を含まないv0.3.1形式JSONも読み込めます。

- 統合読込: 現在のカード理解度を維持
- 学習状態置換: カード理解度を未設定へ置換

置換を選んだ場合は、確認画面に注意文を表示します。

## 問題参照の修復

古いJSONに修正前の数学3問が含まれていても、読込前に次を自動修復します。

- `math-q010`
- `math-q016`
- `math-q024`

これにより、古いバックアップから問題→カード参照切れを再導入しません。

## 確認手順

1. Matrixを「覚えた」にし、Tensorを「曖昧」にする
2. 学習データを書き出す
3. Matrixを「苦手」に変更する
4. 読込方法を「統合読込」にしてJSONを読み込む
5. Matrixが新しい「苦手」のままであることを確認する
6. 読込方法を「学習状態を置換」にする
7. 同じJSONを読み込む
8. MatrixがJSONの「覚えた」へ戻ることを確認する
9. Tensorが「曖昧」のままであることを確認する
10. 問題総数が174問のままであることを確認する
11. 「直前の状態へ戻す」で置換前の状態へ戻ることを確認する

## 注意

- 置換読込は学習状態を戻す操作です。通常の端末間同期には統合読込を使います。
- 自動ダウンロードされた置換前JSONは、確認完了まで削除しないでください。
- 読込処理は端末内で完結し、外部へ学習データを送信しません。
