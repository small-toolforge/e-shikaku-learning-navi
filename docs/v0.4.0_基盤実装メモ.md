## カード理解度と間隔反復

各シラバスカードへ次の3段階を追加しました。

| 状態 | 次回復習 |
|---|---|
| 苦手 | 当日 |
| 曖昧 | 翌日 |
| 覚えた | 3日 → 7日 → 14日 → 30日 → 60日 → 90日 |

「覚えた」を続けて選ぶと復習間隔を段階的に延ばします。「曖昧」へ変更すると翌日、「苦手」へ変更すると当日の復習対象へ戻します。

カード画面には以下を追加しました。

- 苦手／曖昧／覚えたボタン
- 次回復習日の表示
- 状態別フィルター
- 本日までの復習対象だけを表示
- 「本日のカード復習」ボタン
- 苦手・曖昧・覚えた・本日復習・未設定の件数表示

記録画面には、カードの本日復習件数と状態別件数を表示します。

## 保存とバックアップ

IndexedDB名とバージョンは変更せず、`eshikaku_v1`の`meta`ストアへ`cardProgress`として保存します。DBスキーマの更新を伴わないため、既存端末との互換性を維持します。

JSONバックアップへ以下を追加しました。

```text
cardProgress
  cardId
  state
  stage
  intervalDays
  dueAt
  lastReviewedAt
  updatedAt
  reviewCount
```

- 旧バックアップに`cardProgress`がなくても読み込み可能
- 統合読込では`updatedAt`が新しいカード状態を採用
- 直前状態への復元ではカード理解度も置換
- 学習履歴の全削除では、回答履歴・問題SRS・カード理解度を同一トランザクションで削除
- 復元用スナップショットにもカード理解度を含める

問題用SRSとカード用間隔反復は保存先と用途を分離しています。問題は正誤回答を基準にし、カードは本人の自己評価を基準にします。
