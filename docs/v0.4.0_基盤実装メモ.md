## v0.4.0-dev.17 受け入れ結果の保存

実行時セルフチェック後に「結果JSONを保存」を有効化します。Windows、iPhone、Androidで同じ形式の受け入れ記録を残せます。

保存内容:

- アプリ表示版
- 実行日時
- 13項目のOK／NGと詳細
- 端末種別
- ブラウザ／ホーム画面PWA
- 表示領域と画面サイズ
- オンライン状態とSecure Context
- 確認したURL
- カード・問題・図解の件数
- IndexedDB名とバージョン

保存しない内容:

- 回答履歴
- 問題の復習予定
- カード理解度
- JSONバックアップの内容

ファイル名:

```text
E資格学習ナビ_v0.4.0-dev.17_受け入れ結果_YYYYMMDD_HHMMSS.json
```

JSONスキーマ識別子は`eshikaku_acceptance_result_v1`です。セルフチェックが完了するまで保存ボタンは無効です。NGがある結果も障害記録として保存できます。

## v0.4.0-dev.16 参照修正

実行時セルフチェックで検出された、数学3問の「問題→カード参照」切れを修正しました。

| 問題ID | 修正前の参照切れ | 修正後 |
|---|---|---|
| `math-q010` | `compare-1-2-variance-std` | `term-1-2-variance`、`formula-1-2-variance` |
| `math-q016` | `compare-1-2-kmeans-gmm` | `term-1-2-gmm`、`term-2-1-kmeans` |
| `math-q024` | `formula-1-3-mutual-information` | `term-1-3-mutual-information`、`term-1-3-conditional-entropy` |

参照修復ファイルは、問題データの読込み後かつカードへの相互リンク生成前に実行します。

```text
questions-01-math.js
  ↓
question-reference-repair.js
  ↓
question-links.js
```

既にSeed版7を利用している端末にも反映するため、Seed版8で上記3問だけを上書きします。回答履歴、問題SRS、自信度、誤答原因は別ストアに保存されるため維持します。

| 状態 | 次回復習 |

| 苦手 | 当日 |
| 曖昧 | 翌日 |
