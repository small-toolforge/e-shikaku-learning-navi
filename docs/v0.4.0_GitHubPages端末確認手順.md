# v0.4.0 GitHub Pages端末確認手順

## 目的

Windows 11で受け入れ済みの`v0.4.0-dev.20`を、iPhoneとAndroidのホーム画面PWAおよびオフライン環境で確認します。

## 検証用ブランチ

```text
release/v0.4.0-rc.1
```

このブランチは、Windows受け入れ済みのコミット`538223973db24a3c98ab2510c501ead10ae623ac`から作成した固定スナップショットです。

`main`は`v0.3.1`のまま変更していません。

## 公開時の注意

GitHub Pagesは1リポジトリにつき1サイトです。公開元を検証用ブランチへ変更すると、既存のGitHub Pages URLでは一時的に`v0.4.0-dev.20`が表示されます。

この検証用ブランチに、回答履歴、受け入れ結果JSON、個人情報、機密情報は含めません。学習履歴は各端末のIndexedDBへ保存され、リポジトリへ送信されません。

## GitHub Pages公開元の変更

GitHubリポジトリで次の順に操作します。

1. `Settings`
2. 左側の`Pages`
3. `Build and deployment`
4. `Source`を`Deploy from a branch`
5. Branchを`release/v0.4.0-rc.1`
6. Folderを`/(root)`
7. `Save`

公開URL:

```text
https://small-toolforge.github.io/e-shikaku-learning-navi/
```

数分待ってから、カード画面または記録画面で次を確認します。

```text
v0.4.0-dev.20
```

旧版が表示される場合は、ブラウザの通常再読み込み、PWAの終了・再起動を行います。学習データを消す必要はありません。

## iPhone・オンラインPWA確認

1. Safariで公開URLを開く
2. 画面が`v0.4.0-dev.20`であることを確認
3. 共有メニューから`ホーム画面に追加`
4. Safariタブを閉じる
5. ホーム画面アイコンから起動
6. `記録`→`実行時セルフチェック`
7. 確認プロファイルを`ホーム画面PWA`にする
8. セルフチェックを実行

期待結果:

```text
18/18項目 すべてOK
```

確認後、結果JSONを保存します。

想定ファイル名:

```text
E資格学習ナビ_v0.4.0-dev.20_iPhone_ホーム画面PWA_受け入れ結果_YYYYMMDD_HHMMSS.json
```

## iPhone・オフライン確認

1. オンラインPWA確認を完了
2. PWAを終了
3. 機内モードをON
4. Wi-Fiも通信できない状態にする
5. ホーム画面アイコンからPWAを再起動
6. `記録`→`実行時セルフチェック`
7. 確認プロファイルを`PWA・オフライン`にする
8. セルフチェックを実行

期待結果:

```text
19/19項目 すべてOK
```

追加確認:

- 1～5章のカードを検索できる
- 図解16件を切り替えられる
- 15分集中モードを開始できる
- カード理解度を変更できる
- PWAを再起動してもカード理解度が残る

確認後、結果JSONを保存します。

## Android・オンラインPWA確認

1. Chromeで公開URLを開く
2. 画面が`v0.4.0-dev.20`であることを確認
3. `アプリをインストール`または`ホーム画面に追加`
4. Chromeタブを閉じる
5. ホーム画面アイコンから起動
6. `記録`→`実行時セルフチェック`
7. 確認プロファイルを`ホーム画面PWA`にする
8. セルフチェックを実行

期待結果:

```text
18/18項目 すべてOK
```

確認後、結果JSONを保存します。

## Android・オフライン確認

1. オンラインPWA確認を完了
2. PWAを終了
3. 機内モードをON
4. Wi-Fiも通信できない状態にする
5. ホーム画面アイコンから再起動
6. 確認プロファイルを`PWA・オフライン`にする
7. セルフチェックを実行

期待結果:

```text
19/19項目 すべてOK
```

カード、問題、図解、カード理解度の保存も確認します。

## 検証終了後

重大な不具合が見つかった場合:

1. PagesのBranchを`main`へ戻す
2. Folderを`/(root)`のまま保存
3. `work/v0.4.0-atlas-foundation`で修正
4. 新しいRCブランチを作成して再確認

iPhone・Androidとも合格した場合:

1. 受け入れ結果JSONを確認
2. Draft PR #9をReady for reviewへ変更
3. 最終基盤検査を実施
4. PR #9を`main`へマージ
5. PagesのBranchを`main`へ戻す
6. 正式版`v0.4.0`の表示と既存学習データ保持を確認

## ロールバック

`main`へ戻すことで、現在の安定版`v0.3.1`へ戻せます。Pagesの公開元変更はIndexedDBの学習データを削除しませんが、公開元を切り替える前に学習データJSONを保存しておきます。
